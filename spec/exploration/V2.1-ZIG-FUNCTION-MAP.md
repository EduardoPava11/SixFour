# SixFour V2.1 -> Zig function map

2026-06-29. This map sets every relevant `Native/kernels.zig` (and `synth.zig`) kernel against the five V2.1 inputs defined in `spec/exploration/V21Definitions.hs` and `spec/exploration/V2.1-PROBABILISTIC-FIELD.md`. V2.1 is the **pre-collapse distributional lift of V2**: per voxel the payload becomes three energy curves `Curve = Level -> Energy` (`nLevels = 256`, `p ~ exp(-E)`); the GIF89a byte is `collapse = argmin_level E`; and `collapse . curve == the sRGB byte V2Latent.encodeBoundary already consumes`. No locked V2 decision is reopened, so the entire post-collapse byte path is untouched in shape.

---

## 1. Executive summary

| Verdict | Count | What it means |
|---|---|---|
| accepts-as-is | 19 existing kernels | Run unchanged; for curves they gain only an outer per-level loop. |
| needs-signature-change | 4 existing kernels | Scalar-per-voxel output/input must grow an `nLevels` dimension. |
| off-path, forces a new kernel | 2 existing kernels | `s4_linear_to_oklab_q16`, `s4_palette_oklab_to_srgb8`, the OKLab cube-root path does not commute with the delta and is not on the curve spine; each forces a new replacement kernel. |
| irrelevant | 8 existing kernels | Display/look, global-palette, and post-collapse board paths, off the V2.1 field entirely. |
| needs-new-kernel | ~5 distinct new kernels | SIMT bin creation, curve->prob (Boltzmann), collapse (argmin), curve-delta-opponent, per-frame palette-delta. (The clusters flag these ~11 times because each independently rediscovers the same missing kernels.) |

**Single most important finding:** the reversible lift spine is **colour-agnostic**. Every Haar/octant kernel, `s4_octant_lift`/`s4_octant_unlift`, `s4_rgbt_lift_quad`/`s4_rgbt_unlift_quad`, `s4_cube_lift_level`/`s4_cube_unlift_level`, `s4_haar_split_level`/`s4_haar_join_level`, `s4_haar_analyze`/`s4_haar_reconstruct`, `s4_haar_level_nodes`, is a pure per-element integer lift, so it **lifts to curves for free** by driving it with a caller loop over the 256 curve levels. The 2x2x2 -> 1 coarse + 7 residual spine (contract #5), the x,y isotropic spatial reduction (contract #4), and the temporal reversible lift all already exist and need no signature change. The genuinely NEW Zig work is exactly three things, none of which has any analogue in the file: **SIMT bin-creation** (contract #1), **collapse = argmin_level energy** (contract #2), and the **per-frame palette-delta scalar** (contract #4), plus a thin curve-delta+opponent kernel for contract #3.

One reconciliation hazard: `V21Definitions.liftOct8` uses coarse = sum-of-8 / residual = child-minus-first, whereas the Zig `s4_octant_lift` (kernels.zig:857) is the average-lineage separable floor-Haar S-transform. The Zig form is the one to keep (volume-preserving, stays within +/-B; a sum-DC over 256-level energy would breach `SUBSTRATE_BOUND`); the exploration `liftOct8` is the stand-in.

## 3. New kernels V2.1 forces (proposed C-ABI signatures)

All follow the existing `s4_*` C-ABI: `[*c]const T` inputs, `[*c]T` outputs, `i32` return code, explicit scratch where needed. Quantisation of the energy domain (f64 vs Q-int) is open Q5; sketches show the f64 capture form with a Q16 sibling where the spine demands it.

```c
// (1) SIMT BIN CREATION, contract #1, lawBinIsSimtFlat
//     One thread fills one (x,y,t) bin with three length-nLevels energy curves.
//     NO neighbour reads. Output is the per-voxel curve field every lift consumes.
i32 s4_v21_make_bins(
    const f16  *field_src,     // raw capture / scene source, voxel-indexed
    i32         frame_count,   // = 64
    i32         side,          // = 64  (side*side bins per frame)
    i32         n_levels,      // = 256 (open Q1: 256 vs 64)
    i32         k_curvature_q16,// curve sharpness/temperature (detail axis)
    f64        *out_curves);   // [frame_count*side*side*3*n_levels], slab per bin
//     Each thread writes E(level)=k*(level-mu)^2 over 0..n_levels-1 for R,G,B.

// (1b) SYNTH sibling, corpus/fixture engine, deterministic seed-reproducible
i32 s4_synth_curve_burst(
    u64 seed, i32 mode, i32 frame_count, i32 side, i32 n_levels
    i32 k_curvature_q16, i32 detail_q16
    f64 *out_curves);          // same shape; mu = the existing lattice+bilerp+grain core

// (2) CURVE -> PROBABILITY (Boltzmann), contract #1, prob/partition
i32 s4_v21_curve_to_prob(
    const f64 *curve,          // [n_levels] energies for ONE channel
    i32        n_levels
    f64       *out_pmf);       // [n_levels]; out[l]=exp(-curve[l])/Z, Z=sum exp(-curve)

// (3) COLLAPSE = argmin_level energy, contract #2, lawCollapseIsArgminEnergy
//     The seam between V2.1 curves and the V2 byte path. Lowest-index tie-break
//     cloned from nearestCentroidQ16 (kernels.zig:~1305) strict-< discipline.
i32 s4_v21_collapse(
    const f64 *curves,         // [p*3*n_levels] for a whole frame, or one bin
    i32        p,              // pixel count (side*side)
    i32        n_levels
    u8        *out_rgb);       // [p*3] sRGB bytes = collapsePixel; feeds encodeBoundary

// (4) CURVE DELTA + OPPONENT, contract #3, lawOpponentCommutesWithDelta
//     6-neighbour von Neumann stencil; opponent applied to the per-level delta.
i32 s4_v21_opponent_delta(
    const f64 *bin_v1,         // [3*n_levels]  (binR,binG,binB) of voxel 1
    const f64 *bin_v2,         // [3*n_levels]  of the +/-1 neighbour
    i32        n_levels
    f64       *out_lab);       // [3*n_levels]  (dL,da,db) per level
//     dL=dR+dG+dB, da=dR-dG, db=dR+dG-2dB ; pure integer-linear, NO cube root.

// (5) PER-FRAME PALETTE DELTA scalar, contract #4, axisWeight T = PaletteDelta
i32 s4_v21_palette_delta(
    const i32 *palette_a_q16,  // frame t   palette, k entries (OKLab Q16)
    const i32 *palette_b_q16,  // frame t+1 palette, k entries
    i32        k
    i32       *out_delta_q16); // scalar; 256-colour-set distance (L1/EMD or
                               // sum|massA-massB| via s4_board_mass_q16 signatures)

// (6) PER-LEVEL OCTANT LIFT WRAPPER, convenience over the unchanged spine.
//     NOT strictly new math (s4_octant_lift does the work); a batched driver so
//     the per-level loop lives in Zig rather than the Swift/Python caller.
i32 s4_v21_octant_lift_curve(
    const i32 *octant_curves_q16, // [8*n_levels] the 8 octantVoxels' one-channel curves
    i32        n_levels
    i32       *out_coarse_q16,     // [n_levels]
    i32       *out_residuals_q16); // [7*n_levels]
//     for l in 0..n_levels-1: gather child[w][l] -> s4_octant_lift -> (coarse,7).
//     Keep the Zig average-lineage form (not liftOct8's sum-DC; that breaches SUBSTRATE_BOUND).
```

`s4_v21_curve_to_prob` is only needed where the trainer wants the explicit PMF/entropy (the `sExpand` reading); the byte path needs only `s4_v21_collapse`, which can scan energies directly without normalising.

## 4. Reuse wins, what lifts for free, and why

The reversible spine is **representation-agnostic, pure-integer, per-element**. Nothing in it knows whether a cell is "a colour" or "an energy at level l"; it only adds, subtracts, and divides by units of Z[1/2]. So driving any of these with an outer `for l in 0..255` loop turns a scalar lift into a curve lift with **zero signature change**:

- **The octant spine (contract #5)** is already `s4_octant_lift`/`s4_octant_unlift` (kernels.zig:857/887), built on `s4_rgbt_lift_quad`/`s4_rgbt_unlift_quad` (816/836). `lawOctantIsNeighbourhood` proves the 8 octant children ARE the +/-1 stencil, so the encode-delta stencil and the energy-lift stencil are the same discrete-geometry object, the lift the curves need is literally already shipped. Reversibility (`/8` exact) = `lawOctantLiftReversible`; equal children -> zero residuals = `lawSBarredOnFloor` (S barred on the reversible floor).
- **The x,y isotropic spatial reduction (contract #4, weight=1)** is exactly an unweighted separable 2-D Haar = `s4_cube_lift_level`/`s4_cube_unlift_level` (917/956). No weighting belongs here; anisotropy lives only on the t axis.
- **The temporal reversible lift** is `s4_haar_split_level`/`s4_haar_join_level` (988/1020). It STAYS unweighted and byte-exact, the palette-delta weight is applied when COSTING the t-detail (in `neighbourCost`), never inside the bijection, or it stops being reversible.
- **The full (L,a,b) pyramid** is `s4_haar_analyze`/`s4_haar_reconstruct` (595/649). After the opponent transform the per-level payload is exactly an interleaved (L,a,b) triple, this kernel's native layout. `lawOpponentCommutesWithDelta` is what licenses feeding it deltas and recovering opponent colour losslessly.
- **The whole post-collapse byte tail**, `s4_dither_frame` (1339), `s4_significance_fill` (1462), `s4_quantize_frame` (343), `s4_gif_assemble` (1805), `s4_gif_decode` (2066) and its scratch helper (1927), is untouched: V2.1 sits ABOVE the encode boundary, and `collapse . curve` hands these exactly the scalar sRGB/OKLab they already accept. `s4_quantize_frame`'s lowest-index argmin tie-break is even the reference to clone into the new collapse kernel (same discipline, different axis: 3-D centroid set vs 1-D curve).
- **`s4_widen_half_to_q16` (251)** is per-element with an element count `n`, so widening a bin's 3x256 curve is one call with `n=768`.

In short: contracts #4 (x,y linear, the spatial/temporal lifts) and #5 (the octant spine, S/K, read-depth) are already realised in Zig; only the per-level driver loop is new, and even that can be hidden behind the optional `s4_v21_octant_lift_curve` wrapper.

## 5. Ordered Zig build order (gated by gate.sh / golden-vector discipline)

Each step is gated the SixFour way: the law/golden lives in the Haskell spec first, `cabal test` pins it, `cabal run spec-codegen` emits the contract, then `cd Native && zig build test` proves the Zig matches byte-for-byte. Promote `V21Definitions.hs` into a gated `Spec.V21*` module before any Zig export ships (it is currently exploration-tier, NOT in cabal/Map/gate).

1. **Promote the contract + resolve the f64-vs-Q16 question (open Q5).** Wire `V21Definitions` into a gated spec module so `lawBinIsSimtFlat`, `lawCollapseIsArgminEnergy`, `lawCollapseIsMode`, `lawOpponentCommutesWithDelta`, `lawOctantLiftReversible` become `cabal test` gates. Decide the energy quantisation rule (the 256-level integer curve, tied to the /3 guard), this fixes every new kernel's payload type. **No Zig until this is green.**

2. **`s4_v21_collapse` first (contract #2).** It is the seam to the existing byte path and the lowest-risk new kernel (a 1-D argmin cloning `nearestCentroidQ16`'s tie-break). Golden: a curve field whose `collapse` byte output must equal the existing pipeline's starting sRGB bytes, so `collapse . curve` is verified against the live encoder. This de-risks everything downstream because the entire accept-as-is tail is now provably reachable.

3. **`s4_v21_make_bins` + the synth sibling `s4_synth_curve_burst` (contract #1).** The embarrassingly-parallel bin producer. Golden: `lawBinIsSimtFlat` (every channel vector length 256) + a determinism golden on the synth seed. Migrate `s4_synth_burst`/`s4_synth_burst_detail` (synth.zig:226/243) to the curve output here, re-attaching the existing `synth -> s4_quantize_frame -> s4_palette_oklab_to_srgb8 -> s4_gif_assemble` test chain AFTER a collapse step. The `detail==DETAIL_Q16` byte-identity contract is re-pinned against the new canonical curve burst.

4. **The per-level spine drivers (contract #5, #4).** No new math, add the `s4_v21_octant_lift_curve` wrapper (and, if wanted, per-level cube/haar drivers) over the unchanged `s4_octant_lift`/`s4_cube_lift_level`/`s4_haar_*`. Golden: round-trip a curve field through lift+unlift and assert byte-exact recovery per level (`lawOctantLiftReversible` extended over `nLevels`). Reconcile the `liftOct8` sum-DC vs Zig average-lineage discrepancy HERE (keep the Zig form; assert it stays within `SUBSTRATE_BOUND`).

5. **`s4_v21_opponent_delta` (contract #3).** Per-level 6-neighbour delta + linear opponent. Golden: `lawOpponentCommutesWithDelta` ported to Zig (opponent-before == opponent-after subtraction, byte-exact). This replaces the off-path `s4_linear_to_oklab_q16` on the curve path; the icbrt/M1/M2 machinery is left untouched for the still-live post-collapse OKLab path.

6. **`s4_v21_palette_delta` (contract #4).** Last because it depends on a settled set-distance definition (open Q3). Fed by `s4_gif_decode` (2066) out_palettes -> `s4_srgb8_to_oklab_q16` (1906); can reuse `s4_board_mass_q16` (1109) to turn each frame's palette into a comparable 16^3 mass signature, then sum `|massA-massB|`. Golden: `lawXyLinearTimeWeighted` (pd=0 -> free t-step; pd>1 -> t costs more than the same spatial colour change). Wire it as `axisWeight(T)` in the curve-delta cost, not inside any reversible lift.

7. **`s4_v21_curve_to_prob` (optional, contract #1 entropy).** Only if the trainer needs explicit PMF/entropy for the `sExpand` (S=expand) reading. Golden: `lawCollapseIsMode` (argmin energy == argmax prob == mode).
