## 3. The five paths

"Confluent modulo the gate" = SUGGESTIVE, name FORCED (it is coarse-projection invariance / bisimulation-modulo-observation, not Church-Rosser reconvergence). PonderNet-as-reduction-count = SUGGESTIVE with a type fault.
**Call. GO-WITH-CONDITION (narrow).** Ship only `V2SkiRefine.hs` proving exactly two laws: anchored refinement IS an S-share, floor IS a K-const. Drop `V2CombinatorFragment` and `V2PonderReduction` as specified. This earns the S-name precisely where detail = f(coarse) and refutes it where detail is free.

## 4. The user's pyramid against the existing spine

```
   64^3  (PIVOT / ANCHOR, = 8^6, a complete capture, RungPivot.pivotSide)
    |  DOWN: SuccessiveRefinement.split 2 6      HELD / within-capture / EXACT
    v        (no invention; lawRefineRoundTrip: refine . split == id)
  16^3 + r_down        <- [32^3 latent, NEVER surfaced, lawIntermediateNeverSurfaces]
    |  refine 6 replays r_down                   bit-exact recovery of the 64^3
    v
   64^3 + r_up         <- [128^3 latent, NEVER surfaced]
    |  UP: octantLift / pairedLift               INVENTED / beyond-capture
    v        gated by RedownsampleGate.passesGate 2 (re-pools to 64^3 anchor)
  256^3   = GIF89a (per-frame palettes = VALUE, index planes = CONTENT)
```

- **The byte-exact FLOOR** = all residuals zero = `synthBeyond . synthBeyond` from the derived 16^3 = deterministic nearest-neighbour upsample with NO invention. Realized arithmetically: `reenterQ16 0 == 0`, no sentinel branch. This is the immutable root every path and every learned weight sits ABOVE.
- **r_down is held-exact capture data** (recovers 64^3 bit-for-bit). **r_up is the learned super-res surplus** (the W_DETAIL term the 5h run pushed to +59.6%). They share the OPERATOR shape (self-similar, `levelsPerStep = 2`) but NOT epistemic status; "self-similar" must not smuggle in "same kind of data."
- **The /3 Eisenstein hazard**: RGB <-> (luma, chroma) is integer-exact only because for real RGB the three /3 numerators are divisible by 3. An arbitrary (luma,a,b) lattice point from a learned residual or a chroma nudge can floor silently and break the bijection. Two compatible resolutions on the table: (a) store all residuals per-channel in RGB so the inverse is never on the live path (Path 3 discipline); (b) the index-3 sublattice Lambda = {l ≡ a+b mod 3} makes exact inversion a STRUCTURAL membership property and any residual/nudge projects onto Lambda before commit (Path 4 theorem). Path 4 is the stronger structural answer; Path 3 is the cheaper enforced-discipline answer. V1's pure dyadic Z[1/2] does not need either.

## 5. Path-curation as one unifying object

The three subsystems are one object viewed three ways:

- **Paint field** (`CellNudge.CellBudget`, 16^3 control grid x 9 channels) is WHAT the user touches: a per-region detail budget, clamped non-negative, zero == floor.
- **Residual pyramid** (the spine above) is WHAT the budget gates: each painted cell governs its 4096-leaf twiceness subtree, biasing how much r_up invention survives in that region. Anchored `PairedResidual` (r = r(v), referenceable token) is the natural backing for a reusable, committable path.
- **Reduction-sequence-as-path** (Path 5 `Path = [Edit]`) is the HISTORY: an ordered fold onto the floor root, undo = prefix truncation, branch/compare via a rose-tree zipper, every Paint-path re-pooling to the anchor by construction. The mechanical link from a painted budget to deeper refinement is `PonderHaltDistribution.lawLowerHaltRefinesMore` (more paint -> lower halt prob -> more steps), and the data-side knob is Path 2's reductionLength prior.

The honest unifying statement: a curation path is a left-fold of local non-invertible edits over the immutable byte-exact floor, where disjoint edits commute (trace monoid), Paint edits stay in the gate null space, and the SKI/reduction framing is point-free NOTATION for that fold, not structural power.
