# SixFour Research — COLOR-AS-MEASURE & PALETTE: optimal-transport barycenters, the Lloyd-Max MSE ceiling, perceptual uniformity (OKLab→Oklch+), color-harmony/beauty, dithering + temporal super-resolution — what is a good palette mathematically, and where can a learned model beat deterministic maximin perceptually?

> Deep-research output (cited, adversarially verified) from the 2026-06-18 canonical-core workflow.
> Informs `docs/SIXFOUR-CANONICAL-PATH.md`. External claims are design inputs until grounded in-repo.

## Hard constraints on the canonical core (per this axis)
- MSE/ΔE-Euclidean reconstruction is a HARD CEILING (Lloyd-Max): the canonical core must NOT contain any learned component trained against reconstruction MSE. Maximin/Q16 collapse stays as the MSE-optimal P-operator base; learned value lives only in perceptual/temporal/preference objectives.
- The perceptual metric the core OPTIMIZES and EVALUATES against must be spatial (S-CIELAB-class) for stills and temporal (ColorVideoVDP-class) for the 64-frame burst — never bare point-wise ΔE, which structurally hides the dither/flicker wins the product is built on.
- Any optimal-transport step in the core must stay inside a TRACTABLE regime (NP-hardness + 1.0013-inapproximability of general barycenters): either fixed shared support (the 256-leaf σ-symmetric lattice — network-flow tractable) or 1-D/sliced OT along the cube-ladder Haar axes. No exact multi-measure barycenter on-device.
- A perceptual uniformity warp must be an owned, byte-exact, spec-gated transform: an Oklch+-style 3-parameter monotone (L-power + C-Naka-Rushton) realized as a fixed-point Q16 LUT in a new s4_* kernel, with Haskell laws + golden vectors, fitted offline on MLX and shipped as constants. No floating-point net on the device path.
- Harmony/beauty must enter as a CLOSED-FORM owned prior (Ou-Luo over the σ-pair structure), with the A/B Bradley-Terry preference learning only the residual away from that prior — the core may not depend on large amounts of preference data to reach a good baseline.
- All offline learned/perceptual machinery (MS-SWD loss, ColorVideoVDP reward, Oklch+ fitting, Ou-Luo residual learning) is Tier-1 Mac/MLX only and emits CONSTANTS into the Zig core; it must never become an on-device runtime dependency (preserves zero-dep Tier-2 + SIMT bit-agreement).

## Findings
### [high] Lloyd-Max is the provably-optimal SCALAR (1-D) MSE quantizer (centroid + nearest-neighbor-midpoint necessary conditions) but is only LOCALLY optimal (non-convex; init-dependent) and its conditions are exactly k-means' fixed point. For a fixed palette size the MSE floor is therefore real but achievable by deterministic maximin/k-means, so an NN CANNOT beat the MSE objective — the in-repo prior is correct and now externally re-confirmed.
**Implication:** Keep deterministic maximin/Q16 collapse as the MSE-optimal P-operator base. Do NOT train any net against an MSE/ΔE-Euclidean reconstruction loss — it is mathematically ceilinged and will at best tie. The learned layer must optimize a DIFFERENT objective (spatial-perceptual, temporal, or preference), never reconstruction MSE.
**Sources:** https://people.kth.se/~mflierl/EQ2845/08-Quantization.pdf, https://ocw.mit.edu/courses/6-450-principles-of-digital-communications-i-fall-2006/926689aaa62a0315473fa9b982de1b07_book_3.pdf, https://www.sciencedirect.com/topics/computer-science/quantizer-design

### [high] k-means/Lloyd-Max minimize squared error even when run IN CIELAB/OKLab; squared-Euclidean-in-a-uniform-space is NOT the same as a perceptual color-difference metric (CIEDE2000), so the MSE-optimal palette is provably perceptually suboptimal. The gap between argmin-MSE and argmin-perceptual is exactly the room a learned/fitted model can claim.
**Implication:** The perceptual win is a re-WEIGHTING / re-METRIC of the SAME collapse, not a new clustering engine. Bolt a perceptual weight (CIEDE2000/spatial) onto the existing Q16 maximin argmin key as a thin residual on assignment, not a replacement clusterer. This is the smallest, spec-able intervention.
**Sources:** https://arxiv.org/pdf/2406.19520, https://medium.com/@sitianliu_57680/how-to-pick-the-optimal-color-palette-from-any-image-ef1342da8b4f

### [high] Oklch+ (arXiv 2606.05255, 2026) extends Oklab with just THREE fitted parameters — a power function on L and a Naka-Rushton compression on C — and reaches STRESS 29.09 on COMBVD vs CIEDE2000's 29.13 (statistical tie) while Oklab alone is 51.45. It is a tiny fitted monotone transform, not a network, and makes plain Euclidean distance perceptual. Caveat: fitted on sRGB-centered COMBVD; high-chroma/HDR validation is explicitly future work.
**Implication:** This is the single highest-leverage, lowest-risk recommendation. The σ-pair genome already lives in a generator space; add an Oklch+-style 3-param (L-power, C-Naka-Rushton) monotone warp as a NEW owned Zig kernel (s4_oklchplus_warp) applied BEFORE the Q16 argmax key. It is byte-exact-able (fixed-point monotone LUT), spec-able as 3 Haskell laws, golden-gatable, and upgrades every existing maximin collapse to near-CIEDE2000 perceptual quality with ZERO net and zero retraining. Fit the 3 params offline on MLX (Tier 1), ship constants to Zig (Tier 2).
**Sources:** https://arxiv.org/abs/2606.05255, https://arxiv.org/html/2606.05255

### [high] Computing Wasserstein barycenters is NP-hard (curse of dimensionality absent from plain OT), AND it is NP-hard to even APPROXIMATE the restricted-support barycenter within a factor of 1.0013. Fixed-support 2-measure case reduces to a network-flow (tractable); 3+ sparse-support is hard.
**Implication:** Hard ceiling on the 'collapse = exact W2 barycenter of 64 palettes' framing: an exact/near-exact barycenter is NP-hard, so deterministic maximin is the RIGHT engineering choice, not a compromise. Where OT IS wanted, restrict to the tractable regimes: (a) fixed shared support (the 256-leaf σ-symmetric lattice IS fixed support → network-flow tractable), or (b) sliced/1-D OT (the cube-ladder Haar axes give natural 1-D slices). Frame the learned model as PROPOSING a good fixed support that the deterministic flow then fills — never as solving the NP-hard barycenter.
**Sources:** https://arxiv.org/pdf/2101.01100, https://epubs.siam.org/doi/10.1137/21M1390062, https://arxiv.org/pdf/2002.04783

### [medium] MS-SWD (Multiscale Sliced Wasserstein Distance, ECCV 2024) is a TRAINING-FREE, differentiable, mathematically-verified-metric perceptual color-difference measure built on sliced OT; it compares non-local patches of similar color+structure and specifically beats competitors under image MISALIGNMENT (where per-pixel ΔE fails). Usable directly as a loss for color transfer in image/video. (Exact STRESS vs CIEDE2000 not in abstract — medium confidence on the magnitude.)
**Implication:** This is the bridge between 'color-as-measure' (already the repo's framing) and PERCEPTION. Sliced-OT is exactly the 1-D-tractable OT the cube-ladder already affords. Use MS-SWD as the OFFLINE training reward signal for any learned residual (Tier 1 MLX), and as the offline acceptance metric for A/B candidate palettes — it is differentiable so it can shape the LeafOverride δ search. Do not ship MS-SWD on-device; use it to GENERATE the constants/preferences that ship.
**Sources:** https://arxiv.org/abs/2407.10181, https://arxiv.org/pdf/2407.10181

### [high] The perceptual gains of dithering are SPATIAL, not per-pixel: S-CIELAB (Zhang-Wandell) prepends a chromatic/achromatic CSF spatial filter before ΔE and predicts halftone/dither visibility far better than point-wise CIELAB; multiscale vector error diffusion wins specifically on S-CIELAB. The eye integrates a dithered pattern, so a palette that looks worse pixel-wise can look better spatially.
**Implication:** This is the cleanest 'where the NN beats maximin perceptually' answer: the win is in JOINT palette+dither chosen to minimize S-CIELAB (spatial), not ΔE (point). Recommend co-optimizing the existing blue-noise/error-diffusion dither WITH the palette under an S-CIELAB-style CSF-prefiltered objective offline, then shipping the resulting deterministic dither matrix. Evaluate all palette changes with S-CIELAB, never bare ΔE — bare-ΔE evaluation would hide exactly the win you are chasing.
**Sources:** https://www.researchgate.net/publication/2629449_Color_Image_Quality_Metric_S-CIELAB_and_Its_Application_on_Halftone_Texture_Visibility, https://www.caeaccess.org/research/volume3/number7/hassan-2015-cae-651992.pdf

### [high] ColorVideoVDP (2024) is a fully-differentiable full-reference video metric with sustained+transient achromatic and R-G/Y-V chromatic temporal channels (castleCSF) that predicts FLICKER and temporal dithering visibility — things frame-averaged per-frame metrics structurally cannot see.
**Implication:** For the 64-frame burst → GIF / temporal-super-resolution axis, temporal palette flicker is the failure mode and it is INVISIBLE to per-frame ΔE/MSE. Use ColorVideoVDP offline (Tier 1) as the temporal-consistency reward when collapsing 64 per-frame palettes to one (or to a McCann/displacement-interpolation geodesic). This justifies the existing 'palette = measure, motion = displacement residual' memory: score the geodesic with ColorVideoVDP, ship the deterministic result.
**Sources:** https://arxiv.org/html/2401.11485v2

### [high] Ou-Luo (2006) gives a quantitative two-color harmony model in CIELAB: harmony = sum of four factors — chromatic difference, lightness difference, lightness SUM (higher lightness more harmonious), and a hue effect (light-yellow component raises harmony); same-hue/same-chroma pairs harmonize. It is a closed-form CIELAB regression, not a net.
**Implication:** Harmony/beauty is the one axis MSE/ΔE provably ignore — this is where the A/B (Bradley-Terry) reward earns its keep. Use Ou-Luo as a closed-form, owned, spec-able PRIOR over palette PAIRS (the σ-pair genome is literally a pair structure — exploit this). Initialize the θ=770-D taste vector / LeafOverride search with Ou-Luo harmony as a smoothness prior so A/B preference data only has to learn the RESIDUAL away from a known-good harmony baseline, drastically cutting preference-data needs.
**Sources:** https://www.researchgate.net/profile/Ming-Luo-13/publication/229906278_A_color_harmony_model_for_two-color_combinations/links/5c0c957192851c39ebde1ea2/A-color-harmony-model-for-two-color-combinations.pdf, https://onlinelibrary.wiley.com/doi/abs/10.1002/col.20208

### [medium] Learned few-color quantizers (CQFormer, ColorCNN) DO add value over k-means/median-cut, but specifically by preserving PERCEPTUAL/SEMANTIC STRUCTURE (CQFormer adds a perceptual-structure loss anchored to World-Color-Survey naming grids; pure-MSE ColorCNN scatters nearby colors to different indices) — the win is structure/naming, evaluated on downstream task accuracy, NOT reconstruction MSE.
**Implication:** Confirms the abandoned grayscale-L supervised look-net failed for the RIGHT reason (it chased reconstruction, which is ceilinged). If any learned palette layer is revived, its loss must be perceptual-structure / naming-consistency (WCS grid, already cited in repo memory for WCS color categories), not L-reconstruction. But given CQFormer is a full transformer and the repo wants zero-dep Tier-2, prefer the Oklch+/Ou-Luo closed-form route over reviving a net.
**Sources:** https://arxiv.org/abs/2212.03434, https://openreview.net/pdf?id=z2kUV2XQBT2

## Risks
- Oklch+ is brand-new (2026) and fitted on sRGB-centered COMBVD with explicitly-deferred high-chroma/HDR validation; adopting its exact 3 params risks miscalibration outside sRGB. Mitigation: adopt the FORM (L-power + C-Naka-Rushton) but RE-FIT the params on data matching SixFour's actual capture gamut, and keep the warp behind a kill-switch like the existing GLRM one.
- MS-SWD/ColorVideoVDP magnitudes vs CIEDE2000 were not pinned to exact STRESS numbers from the abstracts I could load (PDF binary / abstract-only). The DIRECTION (sliced-OT and spatio-temporal metrics beat point ΔE on misaligned/temporal data) is well-supported, but the size of the win is medium-confidence; verify on the full ECCV/paper PDFs before committing training budget.
- Temporal perceptual optimization (ColorVideoVDP) is differentiable but heavy; using it as a training reward is fine offline, but there is a temptation to approximate it on-device — that would break zero-dep + bit-agreement. Hold the line: ship only the constants/dither it produces.
- Ou-Luo is a TWO-color model from 2006 on CRT/CIELAB with a medium-gray surround; generalizing it to a 256-leaf palette and modern P3/OLED viewing is an extrapolation. Use it as a soft prior over PAIRS (matching the σ-pair genome), not as a hard 256-way harmony objective, and let A/B data correct it.
- General Wasserstein-barycenter NP-hardness could be mis-cited as forbidding ALL OT in the core; the nuance is that fixed-support and 1-D/sliced cases ARE tractable. Misreading this could cause over-conservative abandonment of the (correct, tractable) sliced-OT collapse the repo already uses. Keep the tractable-regime carve-out explicit in any spec note.
- Reviving any learned palette net (CQFormer-style) risks repeating the abandoned grayscale-L failure if its loss drifts back toward reconstruction; the closed-form Oklch+ + Ou-Luo route is strictly lower-risk and zero-dep-compatible, and should be exhausted before any net is reconsidered.
