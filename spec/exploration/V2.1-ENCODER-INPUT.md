## 0. The answer in one paragraph

The encoder input is the **ground-state-centered integer energy field, presented mode-relative**, decimated onto the locked 64-token octant lattice and cast to `float32`. Stored field is integer everywhere (`EnergyQ16 = Int`, `V21Field.hs:93`); the encoder is float; the single named seam is one read-time cast `ℤ ↪ ℝ`. The learnable quantity is **not** counts (carry the nuisance scale `N`), **not** mass (`countsToMass` divides by the non-unit `N`, `V21Field.hs:224-225`, so it cannot survive the integer floor), and **not** surprisal (`−log p` is transcendental and breaks the order-only re-entry). It is **energy**, presented as `Ê(v) = maxCount − count(v) ≥ 0`,  the per-level excess above the ground state,  and then **reindexed about its own mode** so that the absolute mode coordinate is *withheld* from `x_field`. Energy is the unique face that is integer, linear (so it commutes with the locked opponent transform, `lawOpponentCommutesWithDelta`, `V21Field.hs:286-297`), and order-correct (so deploy re-enters byte-exactly via `argmin`, `lawCollapseIsArgmin`, `V21Field.hs:280-284`). The argmin-redundancy trap is resolved structurally, not dodged: because `x_field` is mode-relative its argmin sits at relative-0 for every voxel, so the absolute mode is **not** a functional of `x_field`,  the GIF, which *is* the mode, supplies the one coordinate the field-as-presented genuinely lacks. Collapse-safety is preserved by keeping the existing no-peek discipline unchanged: the predictor's context is the **coarse + 6 sibling detail bands** of the lifted octant (`lawHeldTargetIsExcludedFromContext`, `JepaData.hs:88-93`), the masked voxel is excluded entirely, and a new no-leak law proves the **vector** target band is not determined by the context modes.

## 2. The ideal encoder input

**Recommended quantity:** the ground-state-centered energy `ê(v) = maxCount − count(v)`, **presented mode-relative** (`ễ`, absolute mode withheld), organized on the depth-2 octant lattice.

**Why this and nothing else** (forced by the locks): counts carry the nuisance scale `N`; mass divides by the non-unit `N` (`V21Field.hs:224-225`) and dies at the floor; surprisal is transcendental and breaks `argmin` re-entry. Energy is integer, **linear** (commutes with `opponentI`, `V21Field.hs:286-297`), and **order-correct** (`argmin` re-entry, `:280-284`). Centering on the ground state costs one ring subtraction and makes the GIF byte the zero of every curve.

**Why mode-relative (the redundancy fix):** the bare centered curve is *not* an admissible encoder input alongside the GIF, because `collapseQ16(centeredEnergy c) = i` (the mode is recoverable from it,  `lawCollapseIsArgmin`, `lawCollapseEnergyIsMode`, `V21Field.hs:280-284,352-354`), so the GIF would be a deterministic function of `x_field` and the two streams would be redundant. We therefore strip the absolute mode coordinate out of `x_field` by reindexing each curve about its own mode (`argmin ễ ≡ relative-0` for every voxel, carrying no absolute-location bit). The GIF then supplies exactly that withheld coordinate,  *where* each spike sits,  and field `+` GIF together reconstruct the absolute centered field. This is the structural non-redundancy: neither stream is a function of the other.

**The trainability fix (decisive):** the input is presented at the **decimated** octant-lattice resolution, never the raw 64-cube. `accumulateHist` box-decimates the fine grid into coarse voxels, keeping each cell's value histogram (`V21Field.hs:251-271`). Decimating the 64³ cube by 16 per axis yields the `4³ = 64` tokens that ARE the locked ViT waist (`HalfwayLatent.hs:60-61`). Per channel this is `64 · 256 = 16384` floats,  kilobytes, not the multi-GB raw `[T,Y,X,3,256]` tensor the naïve reading implies.

**Tensor layout:**
- `x_field : float32 [B, 64, 3, 256]`,  (batch, octant token = 4³ decimated voxel, RGB primary, value-level fibre). Entries `ễ(v) ≥ 0`, **mode-relative**: the argmin sits at relative-0 for every voxel and the *absolute* mode index is withheld (it lives only in `x_gif`). Opponent `(L,a,b) = (R+G+B, R−G, R+G−2B)` (`V21Field.hs:127-130`) is recovered **post-hoc by linearity**, never stored separately. The 256 axis is the totally-ordered value chain (argmin is its only lattice structure).
- **Per-token payload** = the within-voxel off-mode landscape `ễ` (the excitation spectrum about the ground state). **Cross-token relations** the ViT forms = the neighbour energy deltas (`curveDelta = zipWith (−)`, `V21Field.hs:132-134`), exact and opponent-commuting; the `1 coarse + 7 detail` A₇ band split (`liftOctList`, `V21Field.hs:165-169`) is the latent organization over the 64 tokens, not a precomputed channel. Both are "energy," one along the value fibre, one along the spatial spine.
- t-axis anisotropy: `axisWeight pd T = pd`, the per-frame palette delta; x,y weight 1 (`lawXyLinearTimeWeighted`, `V21Field.hs:151-154`).
- `x_gif`,  the realised `(cPalette, cIndex)` (`ConstructionEncoder.hs:88-94`) supplies the **withheld mode anchor** (re-attaches each mode-relative curve to its absolute level) and is injected as **FiLM** `(γ,β)` conditioning on each 512-d field token, not concatenated as another value channel. Critically, for the masked target voxel the anchor stream is *excluded* (see §3), so it never re-supplies the held band's own mode.

## 3. Field + GIF, non-redundant,  and no-leak

**The trap, stated bluntly:** the GIF **is** `collapseQ16(field) = argmin E = the mode` (`lawCollapseEnergyIsMode`, `V21Field.hs:352-354`), and the tie-break is deterministic from the curve (`lawCollapseIsArgmin`, `:280-284`). Feeding the encoder both the **bare centered** field-value and the GIF-value, raw and unmasked, is **provably redundant**,  it is exactly the BYOL-shaped argmin→target copy the manufactured-target lock forbids. **We do not do that.** `x_field` is mode-relative, so its argmin is content-free (always relative-0) and the absolute mode is *not* a functional of `x_field`.

**What the GIF actually adds,  honest accounting.** As raw value data the GIF's value is gauge (`lawPaletteIndexGaugeInvariant`, `ConstructionEncoder.hs:250-263`,  permuting palette slots + remapping indices is byte-identical pixels, so the slot ordering is not data and is canonicalized away). The GIF earns its place through **two non-data roles**:
1. **It is the mode-location coordinate that `x_field` withholds.** `x_field` is mode-relative; it supplies the off-mode *shape* about each spike but not the spike's absolute level. The GIF supplies precisely that anchor. Reconstruction `anchorAt(gifMode, x_field) = centeredEnergy(field)` is the witness that the two streams are complementary, not overlapping. This is the structural non-redundancy.
2. **It is the deploy observable.** On device only sRGB8+index exists; the Q16 field is not recoverable across a shipped GIF (`contractQ16NotRecoverableAcrossGif`, `CaptureFormat.hs:24-30`), so the GIF is the cheap thing the deployed encoder actually sees.

**The masking discipline,  UNCHANGED from the codebase (the only collapse-safe framing).** We keep the existing no-peek architecture exactly, and do **not** substitute a global GIF field for the sibling context:
- **Target:** a held octant detail band, data-manufactured and θ-free: `heldTarget = detailAt m (ocDetail (liftOct cube))` (`JepaData.hs:55-56,79-82`), with `encoderParamCount == 0` (`EncoderFrozen.hs:100-101`) so there is nothing to EMA and no co-evolving target encoder. This manufactured-target / no-EMA half is retained verbatim.
- **Context:** the **coarse + 6 sibling detail bands** of the lifted octant, with the masked band held out,  `lawHeldTargetIsExcludedFromContext` (`JepaData.hs:88-93`): `length (siblingsOf ex) == numBands − 1 = 6`. The predictor reads `featuresB (mbeCoarse ex) (siblingsOf ex)` (`MaskedBandPrediction.hs:218`), **not** a global GIF mode field derived from all voxels including the masked one. The `x_gif` anchor stream therefore *excludes the masked voxel's own mode*; the masked voxel is absent from the context entirely. This keeps us *inside* the existing no-peek law rather than stepping outside it.
- **Why this is leak-free,  a VECTOR no-leak claim, not the scalar `disagree` hand-wave.** The earlier argument cited `disagree = sum − max` (`V21FieldUI.hs:120-125`), a *scalar* UI saliency, and its mode-permutation invariance. That object is the wrong shape: the held target is a *vector* detail band of `liftOct` of the curves. Scalar invariance says nothing about whether the vector target is determined by the context. The correct guarantee is the new law in §8: the held detail band is **not** determined by the context modes,  there exist two octants of curves with identical per-child modes (identical GIF) but different lifted detail bands, so even a context that saw every mode could not copy the target. Combined with `lawHeldTargetIsExcludedFromContext` (the masked band is physically absent from context), prediction is real work, not copying.

## 4. Physics justification

The field is a discrete EBM: each voxel-channel carries a 256-level energy landscape, `p(v) ∝ exp(−E(v))` (Boltzmann), ground state `= argmin E =` the GIF byte. **Honest scope:** the stored energy is the count-complement `E = N − count`, not `−log p`, so `exp(−E) ∝ exp(count)`, not the empirical mass,  the two agree only on the *order* (the ground state), which is all the byte-exact path uses, and the true partition-function reading is quarantined off-path (`V21Field.hs:24-31`). Energy is the additive/extensive face, so the Haar lift averages it exactly and the coarse stays a bounded floored-mean lineage, never a sum-DC that breaches the substrate (`lawOctantCoarseBounded`, `V21Field.hs:313-318`). Centering chooses the GIF as the potential's zero; presenting it mode-relative means the encoder predicts the **excitation spectrum** about the ground state while the ground-state *position* is carried separately by the GIF anchor. Mass (a ratio) and surprisal (transcendental) are physically the wrong observables to floor; energy is the unique integer, linear, order-correct face.

## 5. Discrete-geometry justification

The `1 coarse + 7 detail` split is the root lattice **A₇ = ker Σ**, the densest dim-7 packing; band count `= 7 = rank A₇` is an identity (`lawOctantIsA7`), zero-integral wavelet detail = membership in `ker Σ`. The lift's floored halving `sLift x y = (y + d div 2, d)` (`OctreeCell.hs:131-132`) is a **reversible set-bijection of ℤ⁸**, exactly inverted by `unliftOct` (`lawOctReversible`, `OctreeCell.hs:202`), and `liftOctList` reuses it colour-agnostically (`V21Field.hs:165-169`). The masked target band and 6 sibling context bands are seven coordinates of this lift, which is why excluding one band (the target) from context is a clean lattice operation, not an ad-hoc mask.

**The seam's left-invertibility correction (grafted verbatim).** The prompt's "`/8` is a unit" is **false against the code**: the ring is **ℤ**, `units = [1,-1]`, and `unitInverse 2 = Nothing` (`RefinementSystem.hs:64-76`, `lawNonUnitsHaveNoInverse`). The float encoder's integer preimage is **not** an appeal to unit-division,  it is (a) the exact `ℤ⁸` bijection `unliftOct ∘ liftOct = id`, and (b) the bare `ℤ ↪ ℝ` cast whose left inverse is `round`. This is strictly more correct than the `ℤ[1/2]`-unit gloss and *reinforces* module-theory-not-linear-algebra (the floored lift is bijective but **not** ℤ-linear). The 64 tokens are the depth-2 octant lattice; the 256 levels are a 1-D ordered fibre that `argmin` reads.

## 6. Algebraic-number-theory justification

`opponentI` is a linear `ℤ³→ℤ³` of **determinant 6**, landing in an index-6 sublattice of ℤ³ (`V21Field.hs:127-130`); linearity is precisely the licence to encode deltas and rebuild opponent colour (`lawOpponentCommutesWithDelta`, `:286-297`). Chroma quarter-turns are available as the **Gaussian ℤ[i]** unit `i` (order 4, the wired `GaussianChroma` knob, `RefinementSystem.hs` second base ring), used only as a discrete chroma gauge symmetry,  **not** introduced on the value path. The **Eisenstein** index-3 / `(1−ω)` ideal program is *exploration-only* and stays behind the gate-wall; this design does **not** wire `(r+g+b, r−b, g−b)` as a live featurisation (that would be a colour-ring choice on the live path the M1→M2→M3 sequencing exists to govern, and it conflicts with the wired `opponentI` chroma). The S₂₅₆ palette gauge is invariant theory, not Galois: the realised slot ordering is canonicalised to an orbit representative, not fed as data,  which is also why the GIF's *value* is non-data and only its *mode-anchor* role is admitted in §3.

## 8. Open questions + the first buildable spec laws/goldens

**Open questions.**

2. Mode-relative windowing: fixed `[−255,255]` displacement window vs a compact bounded-support window padded with the off-path sentinel,  the exact reindex convention that makes `anchorAt` a clean left inverse of `modeRelative`.
3. `x_gif` injection: FiLM `(γ,β)` mode-anchor (recommended, respects the gauge) vs token concatenation.
4. Whether to also feed the 7 detail-band curves explicitly as channels, or let the ViT form the A₇ neighbour deltas over the 64 tokens (recommended: let it form them).
5. `PaletteDelta` at the decimated resolution: how the per-frame palette delta is computed once box-decimation has run.
6. The box-decimation factor (16, giving the locked 64-token waist) vs an alternative,  the waist is locked, the factor that fills it is a choice.

**First buildable spec laws (in `SixFour.Spec.V21Field`, gateable now).** Three laws pin the load-bearing claims: non-redundancy (mode is withheld from `x_field`), reconstruction (field `+` GIF recover the field), and **vector** no-leak (the target band is not determined by the context modes).

```haskell
centeredEnergy :: Curve -> Curve
centeredEnergy c = let m = minimum c in map (subtract m) c

-- | DEPLOY-PATH ONLY (NOT the encoder input): centering is a monotone transform, so byte re-entry
--   via argmin is unchanged and the int↪float cast round-trips. This proves deploy byte-exactness;
--   it deliberately does NOT present the bare centered curve to the encoder (that would leak the mode).
lawCenteredEnergyDeployRoundTrips :: [Int] -> Bool
lawCenteredEnergyDeployRoundTrips cs =
  null cs ||
    let e  = countsToEnergy cs
        ce = centeredEnergy e
        i  = collapseQ16 e
    in  ce !! i == 0
        && map (round . (fromIntegral :: Int -> Double)) ce == ce   -- ℤ↪ℝ left inverse exact
        && collapseQ16 ce == i                                       -- centering monotone

-- | THE ENCODER INPUT: mode-relative energy. Reindex the centered curve about its own mode so the
--   argmin sits at relative-0 and the ABSOLUTE mode coordinate is withheld. 'shiftAboutMode' moves
--   level @mode+k@ to relative index @k@ within the fixed window; 'anchorAt' is its left inverse.
modeRelative :: Curve -> Curve
modeRelative c = shiftAboutMode (collapseQ16 c) (centeredEnergy c)

-- | (1) NON-REDUNDANCY,  x_field withholds the mode. The absolute mode is NOT a functional of
--   x_field: two histograms whose shape-about-the-mode agrees but whose modes differ map to the
--   SAME mode-relative input, so the GIF (the mode) is independent information, not a copy.
--   Teeth: any presentation from which @collapseQ16@ recovers the mode (e.g. bare centeredEnergy) fails.
lawModeRelativeWithholdsMode :: [Int] -> Int -> Bool
lawModeRelativeWithholdsMode cs shift =
  null cs ||
    let e   = countsToEnergy cs
        e'  = countsToEnergy (rotateMode shift cs)       -- relocate the spike, keep its shape
    in  collapseQ16 (modeRelative e) == 0                 -- argmin pinned to relative-0
        && modeRelative e == modeRelative e'              -- field alone cannot see WHERE the mode is
        && (collapseQ16 e /= collapseQ16 e')              -- yet the modes (the GIF) genuinely differ
             `implies` True

-- | (2) RECONSTRUCTION,  field + GIF recover the field. Anchoring the mode-relative input at the
--   GIF-given mode reproduces the centered curve. Witnesses complementarity (no overlap).
lawFieldPlusGifReconstructs :: [Int] -> Bool
lawFieldPlusGifReconstructs cs =
  null cs ||
    let e = countsToEnergy cs
    in  anchorAt (collapseQ16 e) (modeRelative e) == centeredEnergy e

-- | (3) VECTOR NO-LEAK,  the held detail band is NOT determined by the context modes (the GIF).
--   There is an off-mode perturbation of the 8 children's curves that preserves EVERY child's mode
--   (argmin), hence leaves the GIF unchanged, yet moves the lifted detail band 'detailAt m'. So a
--   context restricted to modes cannot copy the vector target. Teeth: replace 'liftOct' by anything
--   that depended only on the argmins and the inequality collapses.
lawTargetNotDeterminedByGifModes :: V8Curve -> Int -> Int -> Bool
lawTargetNotDeterminedByGifModes children m bump =
  let modesOf cc = map collapseQ16 cc
      children'  = raiseOffModeMass bump children          -- adds count to a NON-mode level of child 0
      bandOf cc  = detailAt m (ocDetail (liftOctCurves cc))
  in  modesOf children' == modesOf children                -- GIF (every mode) UNCHANGED
      && (bump /= 0) `implies` (bandOf children' /= bandOf children)  -- but the VECTOR target moved
```

`lawTargetNotDeterminedByGifModes` is the law that replaces the old scalar-`disagree` hand-wave: it discharges the leakage worry on the actual object (the vector detail band), and together with `lawHeldTargetIsExcludedFromContext` (the masked band is absent from context) it restores a real no-peek guarantee for the GIF-conditioned design. Until all three above are green, the field+GIF non-redundancy and no-leak claims are treated as unproven; the manufactured-target / no-EMA half (`liftOct` param-free, data-manufactured) is independently sound and retained.

**First cross-tier golden:** `accumulateHist` box-decimation (factor 16, `64³→4³`) matches the Zig `s4_v21_counts_to_energy` kernel byte-for-byte; `collapseQ16 ∘ countsToEnergy` of each decimated cell reproduces the captured GIF byte for that octant token; and `anchorAt(gifByte, modeRelative(energy))` reproduces `centeredEnergy(energy)` cell-for-cell,  closing the Haskell↔Zig↔GIF loop (including the mode-relative reconstruction) before any MLX training.
