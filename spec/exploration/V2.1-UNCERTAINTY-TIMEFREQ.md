# V2.1 and the time-frequency / uncertainty thesis: what is real, what is a category error, and the gateable steer

Steering output for the owner's proposal that the voxel's value/energy spread and the temporal
`paletteDelta` form a Gabor / Heisenberg time-frequency conjugate pair, and that the Zig accumulate
kernel can be raised to "higher-precision continuous packets, R,G,B in 20fps quanta". Produced by a
4-lens + 3-adversary workflow (wf_f75a8180-4c4, 2026-06-30). This doc is honest by design: it separates
what is proven, what is salvageable, and what must not be built.

## Verdict

The LITERAL framing (value/energy spread is Fourier-conjugate to time; `paletteDelta` is its conjugate
momentum; together they obey a Heisenberg uncertainty bound) is a CATEGORY ERROR. All four independent
lenses returned `claimSound = false` (confidence 0.80 to 0.82), and all three adversarial verifiers
refuted it. Three walls stop it, each sufficient alone:

1. NO CONJUGACY KERNEL. A Fourier/Heisenberg pair needs a transform kernel `e^{-i w t}` tying the two
   axes and two NON-COMMUTING operators (Robertson-Schrodinger). Amplitude/value is not the Fourier
   dual of time: a signal can be perfectly time-localized at any amplitude, so value-vs-time obeys no
   uncertainty product. `paletteDelta` (V21Field.hs) is literally `sum |hist1(ch,v) - hist2(ch,v)|`, a
   scalar L1 distance consumed only as the cost weight `axisWeight T = pd`. A scalar cost weight has no
   spectrum, no phase, and no back-action; it is not a conjugate momentum.

2. THE TIME AXIS IS DESTROYED BEFORE ANY TRANSFORM. `accumulateHist` (V21Field.hs, Zig
   `s4_v21_accumulate_hist` kernels.zig:2949) buckets every fine frame by `cti = fti / dt` and COUNTS
   occurrences per value level. This is a many-to-one, order-destroying marginalization; only totals
   survive (`lawHistTotalPreserved`). You cannot have an uncertainty relation over an axis you have
   already integrated out. There is no DFT, no STFT, no Gabor atom, no Zak transform anywhere in the
   pipeline. The only genuine spectral object in the code is the SPATIAL dyadic Haar (the octant lift),
   which is position-vs-log-scale, not temporal and not uniform-Gabor.

3. THE RING FORBIDS THE TRANSFORM ANYWAY. The spine ring is Z with units {+1, -1} only
   (RefinementSystem.hs; 2, 4, 8 are non-units with no inverse). A minimum-uncertainty Gaussian/Gabor
   packet needs `exp` and division by 2/4; a length-64 DFT needs an order-64 root of unity. None exist
   in Z. So even a wanted conjugate transform is barred from the byte-exact path by construction: it
   would move `argmin` and change the GIF byte. The NTT escape (exact over Z/q, q = 65537) produces
   order-destroying mod-q residues that `collapseQ16` (argmin) cannot read, so it is as unusable on the
   byte path as float.

## What is REAL and survives (the two things to build toward)

### Salvage B, on-path, exact, ALREADY BUILT: Z/256 location-vs-shape complementarity

The genuine complementary pair already in the code is:

- the absolute mode LOCATION: the GIF byte, `collapseQ16` = argmin energy, and
- the mode-relative SHAPE: the field, `modeRelative = rotateLeft(collapseQ16) . centeredEnergy`.

`anchorAt` is the exact inverse (`lawFieldPlusGifReconstructs`), and `lawModeIsNotAFunctionOfField`
proves the two halves are jointly complete and NON-redundant. This is the translation leg of Pontryagin
duality on the self-dual value fibre Z/256. It is a COMBINATORIAL permutation complementarity (location
vs shape), not a spectral spreading inequality. Only the translation leg is implemented; the
modulation/DFT leg is barred by the ring, so there is genuine complementarity but no realized
uncertainty inequality and no non-commuting back-action. This is the honest home for the owner's
intuition that "the palette difference is the expression of invertible sums of curves": the invertible
structure is real, it is just combinatorial (a cyclic shift), not Heisenberg.

### The genuine invertible "sums of curves": the integer Haar wavelet packet

The reversible octant lift (`liftOct`/`unliftOct`, OctreeCell.hs) IS an integer-to-integer lifting-scheme
Haar wavelet packet (Sweldens / CDF lifting): `sLift x y = (y + (x - y) div 2, x - y)`, 1 coarse DC + 7
detail bands, the A7 root lattice as the detail kernel. This is the byte-exact discrete COUSIN of a
Gabor frame, the standard construction that reconciles a localization tradeoff with PERFECT
reconstruction over Z, with no roots of unity and no non-unit division. If "sums of curves with an
invertibility" is to have a rigorous name in this codebase, this is it: a dyadic (log-scale)
multiresolution, not a uniform Gabor lattice.

### Salvage A, off-path float diagnostic: a bias/variance coarse-graining tradeoff

De-localizing in time widens the value marginal. If `accumulateHist` retains `ct = ft/dt > 1` temporal
packets instead of one (the kernel already computes `cti` from `fti / udt`, so this is a free integer
reindex, the 64 = 8x8 Zak LAYOUT as a permutation), you can build a joint (time-packet x value) integer
histogram per cell and compute the dispersion product `sigma_t * sigma_v` from its second moments. This
makes "value spread vs time localization" a MEASURABLE statistical covariance (wider temporal window
gives more motion/occlusion mixing gives wider amplitude spread). It is ordinary bias/variance, NOT
`dt . dw`, and it must never re-enter `collapseQ16`. It also costs the locked 4^3 = 64 token waist
(trades tokens for time bins), so it stays a diagnostic until that lock is explicitly revisited.

## The concrete camera-level change (the honest reading of "continuous packets")

"Higher-precision continuous packets" has exactly ONE honest on-path reading, and it is AMPLITUDE
precision, not temporal frequency. In `s4_v21_accumulate_hist` (kernels.zig, the `out_counts[...] += 1`)
and its Metal twin `v21AccumulateHistKernel`, replace the round-to-nearest unit increment with an
INTEGER fixed-point linear-interpolation splat: for a channel value whose scaled position falls between
levels `v_lo` and `v_hi = v_lo + 1` with sub-LSB fraction, pick a fixed integer weight budget `W` (e.g.
256), add `W - w` to `v_lo` and `w` to `v_hi`. Per-sample mass stays constant at `W`, so counts and
`E = N - count` stay integer, `argmin` stays order-only, and the i32 envelope holds
(`W * frames * box < 2^31`; keep the existing `n_levels <= 256` guard). This recovers roughly the 2
sub-LSB bits that `round()` currently discards from the 10-bit sensor and the box average. It enriches
the trainer FIELD only.

"20fps quanta" as a literal temporal-FREQUENCY acquisition change is NOT available on the byte path: the
histogram already marginalized the frame ORDER away, so there is no flicker/motion spectrum to recover
from it. A true temporal-frequency object would need a NEW buffer holding the ordered `[T,Y,X,3]` burst
BEFORE marginalization plus a float DFT (20fps gives Nyquist 10 Hz, resolution 20/64 ~ 0.3125 Hz/bin),
and it can only ever be an off-path diagnostic. Do NOT add a Gaussian/Gabor kernel on-path: soft
smoothing can move the `argmin` and change the shipped GIF byte (byte-destructive, not merely off-path).
