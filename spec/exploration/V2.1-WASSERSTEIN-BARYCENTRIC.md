# V2.1 and Wasserstein Barycentric Coordinates: steering decision (adopt-weaker)

Steering output for the owner's proposal to apply Wasserstein Barycentric Coordinates (Bonneel, Peyre,
Cuturi, "Wasserstein Barycentric Coordinates: Histogram Regression Using Optimal Transport", SIGGRAPH
2016) to the V2.1 probability field. Produced by a 12-agent understand / design / adversarial-verify /
synthesize workflow (wf_f10148d0-6bf, 2026-06-30). Honest by design: proven vs speculative, and what
NOT to build.

## Verdict: ADOPT-WEAKER

SixFour adopts the 1-D closed-form quantile machinery, NOT the paper's entropic Sinkhorn, and NOT
"coordinates as the model's field encoding". All four maximal claims were refuted by the adversarial
phase; their salvage is exactly this weaker program:

1. SHIP NOW (byte-exact, integer, in Zig): upgrade the temporal metric `paletteDelta` from Total
   Variation (L1 of histograms) to true 1-D Wasserstein-1 (L1 of CDFs).
2. TREAT the barycentric coordinates lambda as an optional, gated, trainer-side TRANSMISSION side
   channel that never sources the GIF byte.
3. RESERVE the existing `Spec.Sinkhorn` / `Spec.Barycenter` strictly for a genuine 3-D joint-RGB /
   OKLab palette barycenter, where no 1-D closed form exists.

## Why the paper's Sinkhorn is not needed here

The paper uses entropic Sinkhorn uniformly across 1-D / 2-D / 3-D because in two dimensions and up the
map from weights to barycenter is not linear in any usable coordinate, so it must differentiate through
the solver (its Algorithm 1). SixFour's per-channel value curves are 1-D histograms over 256 ORDERED
levels (`Curve`, `spec/src/SixFour/Spec/V21Field.hs`). In 1-D, optimal transport is textbook closed
form:

- Wasserstein-1 distance = the L1 distance between the two integer CDFs.
- The barycenter = the lambda-weighted average of the quantile (inverse-CDF) functions (McCann
  displacement), linear in lambda.

So on the per-channel curves the entropic apparatus is not merely unneeded, it is strictly dominated:
the repo's own `Spec.Sinkhorn` is O(iterations * n^2) float with exp / logSumExp, symmetric only to
about 1e-4 by its own note, and entropically biased. The integer CDF path is O(n), exactly symmetric,
unbiased, and lives on the byte-exact ring Z.

## The metric upgrade (the win), stated exactly

`paletteDelta` is `sum_{ch,v} |h1(ch,v) - h2(ch,v)|`, the Total Variation. TV is horizontal-blind: a
one-level colour drift and a far colour jump of equal mass score identically (both are "two bins
differ"). True 1-D W1 is `sum_{ch,v} |CDF1(ch,v) - CDF2(ch,v)|` where the CDF is the running cumulative
sum of the histogram over the value levels. Equivalently, insert ONE integer per-channel prefix-sum of
the signed difference before the absolute-value sum. Now the metric charges the GROUND DISTANCE mass
must travel: a 1-level drift costs 1 per unit mass, a 200-level jump costs 200. This fixes a real defect
and discharges the deferred open Q3 (the "EMD / Wasserstein richer metric") ON the byte-exact path with
zero float and zero Sinkhorn.

Precondition, ASSERTED not normalized: W1-as-L1-of-CDFs requires equal total mass between the two
histograms (else the CDFs end at different totals and the tail contaminates). Two k-slot palettes each
have per-channel total k, so equal mass holds by construction; soft cells each total box*w
(`lawSoftHistTotalPreserved`). Normalizing by the total would reintroduce non-unit division and leave
the ring, so the code asserts equal mass rather than dividing.

## The seam: byte-exact vs float-diagnostic

BYTE-EXACT integer, ring Z, ships in Zig: the integer CDF (prefix-sum of the palette / soft-splat
counts, no division); W1 = L1 of the two integer CDFs (exactly symmetric by construction); the integer
quantile function (the sorted multiset on the pre-sorted level grid); the dictionary stored as integer
energy atoms; and the GIF byte via `collapseQ16` = argmin (order-only, exact under any monotone recode).

FLOAT, trainer-only (Tier-1 MLX / numpy), off the shipped byte: the lambda coordinates; the
simplex-NNLS / Frank-Wolfe fit that produces them; the quantile-average and its single re-bin round; W2
and W2^2 (they carry 1/N and sqrt, off-path like Bures); dictionary-learning gradients. No exp, no
Sinkhorn, no gamma blur, no differentiate-through-a-solver. HONEST boundary: in a compact-transmit
configuration float lambda WOULD flow into the byte via reconstruction + argmin, so that configuration
is a NEW lossy precision boundary and must be labeled and gated, not waved through as "the same as the
softmax diagnostic".

One decisive domain correction: `modeRelative` is a CYCLIC `rotateLeft`, so its value axis is a circle
where level 255 is adjacent to level 0. W1 on a circle is a different metric with no simple inverse-CDF
form. So any WBC coordinate fitting must run on the LINEAR un-rotated shape (`centeredEnergy`) carrying
the mode offset as an explicit integer, never on the cyclic `modeRelative`. The metric upgrade dodges
this entirely because it runs on linear palette-value histograms.

## The SIMT / tensor Zig question (the weaker answer)

Do NOT build the S4Tensor-maps-to-MTLTensor abstraction now. The 42 `s4_*` kernels are CPU C-ABI scalar
functions compiled into an aarch64-ios static lib and called from Swift; there is no SIMT in the Zig
core, parallelism lives only in Metal. A POD shape-struct in Zig is a shape-carrier, not a parallelism
abstraction. Metal 4 `MTLTensor` and Core AI `NDArray` are WWDC26 developer-beta, absent from the
simulator SDK, device-only, GA around Sept 2026: pinning a cross-tier C-ABI golden to them now would
freeze a layout against an unseen target (a quantized `MTLTensor` likely carries separate scale planes,
not element strides).

What to actually build, keeping the existing scalar signature style, no descriptor: `s4_v21_wdist1d`
(the metric, first), then if a family emerges `s4_v21_cdf` (a segmented inclusive prefix-sum over the
256-level axis, one lane per flattened segment scanning only its own 256 run, preserving the locked
"one thread fills one bin, no neighbour reads" contract), then `s4_v21_quantile` and `s4_v21_wbary1d`.
A minimal INTERNAL index-decode helper is defensible ONLY after at least three same-shaped cumsum
kernels exist: let the helper follow the kernels, not precede them, and keep it private to the Zig lib.
Future ops become easier to encode because cumsum + segmented-reduce is one reusable primitive shape,
not because of a speculative descriptor.

## Capture to AirDrop to train

The device captures raw integer soft-splat curves (`accumulateHistSoft`) and ships the field plus the
true GIF byte as it already does. The dictionary D of shape [3, K, 256] integer atoms (K about 8 to 16
per-channel quantile atoms) is learned ONCE on the Mac by MLX-autodiffing the linear barycenter
(bilinear in lambda and the atoms, so the gradient is closed form), frozen, and shipped in the bundle.
Per-voxel lambda of shape [64, 3, K] is fit by the convex simplex NNLS in the quantile domain on the
un-rotated `centeredEnergy` shape plus an explicit integer mode offset, and added to the AirDrop bundle
as a `_bary_*.f16.npy` ALONGSIDE the true byte, never replacing it. The 18.9M ViT consumes lambda as an
optional additive readout head constructed last, so a zero weight is bit-identical to the current
trajectory, or it ignores lambda entirely. Core AI is inference-only and never produces the byte;
on-device adaptation, if ever, is MPSGraph, not Core AI training.

## First gateable step (sized like palette_delta / soft-splat)

Module `SixFour.Spec.V21Field`. Function `paletteW1` mirroring `paletteDelta`, inserting one per-channel
integer prefix-sum over the signed diff histogram before the abs-sum. Laws: `lawWDistZeroOnEqual`,
`lawWDistSymmetric`, `lawWDistGaugeInvariant` (slot permutation leaves the value CDF unchanged because
cumsum is linear), `lawWDistStaticTimeFree`, PLUS the load-bearing discriminator `lawWDistChargesDistance`
(a 1-level drift costs strictly less than a far jump of equal mass, the exact defect TV cannot see, while
`paletteDelta` scores them equal). Zig `s4_v21_wdist1d` as a near-clone of `s4_v21_palette_delta` with a
per-channel running accumulator. Golden `v21_wdist1d_golden.json` + `v21_wdist1d_fixture_test.zig`
modeled on the palette-delta fixture. Gate: `cabal test` + `zig build test -Drequire_fixtures=true`.

## What NOT to build, and the real risks

Do NOT build: the paper's entropic Sinkhorn on the per-channel curves; the differentiate-through-solver
unroll; the tensor descriptor pinned to `MTLTensor` / Core AI goldens before those APIs are GA; any
"lambda replaces the field" transmit mode that drops the true byte.

Real risks: dictionary coverage (paper Fig.12) can corrupt a reconstructed byte, so validate
reconstruction PSNR AND a zero byte-flip-rate under argmin on real AirDropped bursts before trusting
lambda; the cyclic-`modeRelative` vs linear-CDF domain mismatch (run WBC on `centeredEnergy` plus an
integer offset, never on the rotated curve); lambda uniqueness needs a full-column-rank dictionary; f16
quantization of lambda must be shown not to move the collapsed byte; equal mass must be asserted not
normalized.

Proven now: the 1-D closed form is exact, integer, and cheaper than Sinkhorn; the W1 metric fixes TV's
distance-blindness. Speculative until measured: any learnability gain from lambda as a ViT input, and
the entire compact-transmit configuration.
